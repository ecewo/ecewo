# Body

By default, ecewo buffers the entire request body in memory (up to 1MB) before calling the handler. Use `ecewo_body_stream` middleware to process the body as it arrives. Body streaming is useful for large file uploads or when memory efficiency matters.

## Buffered Body (Default)

```c
void handler(Req *req, Res *res) {
  const char *body = req->body;
  size_t len = req->body_len;
  ecewo_send_text(res, OK, "received");
}

ECEWO_POST(app, "/upload", handler);
```

> [!NOTE]
>
> To increase the default body limit, set `BUFFERED_BODY_MAX_SIZE` in your CMake.

## Streaming Body

Add `ecewo_body_stream` middleware before your handler:
```c
void handler(Req *req, Res *res) {
  ecewo_body_on_data(req, on_chunk);
  ecewo_body_on_end(req, res, on_complete);
}

ECEWO_POST(app, "/upload", ecewo_body_stream, handler);
```

---

## API

### `ecewo_body_stream`

```c
void ecewo_body_stream(Req *req, Res *res, Next next);
```

Middleware that enables streaming mode. Must be registered before the handler.

---

### `ecewo_body_on_data`

```c
void ecewo_body_on_data(Req *req, BodyDataCb callback);
```

Registers a callback to receive body chunks.

```c
typedef void (*BodyDataCb)(Req *req, const uint8_t *data, size_t len);
```

---


### `ecewo_body_on_end`
```c
void ecewo_body_on_end(Req *req, Res *res, BodyEndCb callback);
```

Registers a callback invoked when the full body has been received.

```c
typedef void (*BodyEndCb)(Req *req, Res *res);
```

---

### `ecewo_body_limit`

```c
size_t ecewo_body_limit(Req *req, size_t max_bytes);
```

Sets the maximum body size in bytes. Returns the previous limit. Default is 10MB in streaming mode, 1MB in buffered mode. Pass `0` to restore the default.

```c
void handler(Req *req, Res *res) {
  ecewo_body_limit(req, 50 * 1024 * 1024); // 50MB
  ecewo_body_on_data(req, on_chunk);
  ecewo_body_on_end(req, res, on_complete);
}
```

---

## Example

```c
typedef struct {
  FILE *f;
  bool write_error;
} UploadCtx;

void on_chunk(Req *req, const uint8_t *data, size_t len) {
  UploadCtx *ctx = ecewo_get_context(req, "upload");
  size_t written = fwrite(data, 1, len, ctx->f);
  if (written != len)
    ctx->write_error = true;
}

void on_complete(Req *req, Res *res) {
  UploadCtx *ctx = ecewo_get_context(req, "upload");
  fclose(ctx->f);
  ctx->f = NULL;

  if (ctx->write_error) {
    ecewo_send_text(res, INTERNAL_SERVER_ERROR, "Write failed");
    return;
  }

  ecewo_send_text(res, OK, "Uploaded!");
}

void upload_handler(Req *req, Res *res) {
  UploadCtx *ctx = ecewo_alloc(req->arena, sizeof(UploadCtx));
  ctx->f = fopen("/tmp/upload.bin", "wb");
  ctx->write_error = false;

  if (!ctx->f) {
    ecewo_send_text(res, INTERNAL_SERVER_ERROR, "Failed to open file");
    return;
  }

  ecewo_set_context(req, "upload", ctx);
  ecewo_body_limit(req, 100 * 1024 * 1024); // 100MB
  ecewo_body_on_data(req, on_chunk);
  ecewo_body_on_end(req, res, on_complete);
}

ECEWO_POST(app, "/upload", ecewo_body_stream, upload_handler);
```

> [!NOTE]
>
> `FILE` is used just to show a body streaming example. You should use [`ecewo-fs`](https://github.com/ecewo/ecewo-fs) for file operations.
