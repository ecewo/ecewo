## Handlers

A handler in ecewo is very similar to Express.js. Handlers take two parameters: `Req *req` and `Res *res` objects.

```c
#include "ecewo.h"

void hello_world(Req *req, Res *res) {
  send_text(res, 200, "Hello, World!");
}
```

## Request Body

`req->body` holds the raw request body as `uint8_t*` (a byte array). `req->body_len` holds its length. A null terminator is always appended, so it is safe to treat as a string when the content is text.

### Text body (JSON, plain text, form data)

Cast `req->body` to `const char *` when the content is text:

```c
#include "ecewo.h"

void print_body(Req *req, Res *res) {
  if (!req->body || req->body_len == 0) {
    send_text(res, 400, "Body required");
    return;
  }

  const char *body = (const char *)req->body;
  printf("Body: %s\n", body);
  send_text(res, 200, "Success!");
}
```

### JSON body

Cast to `const char *` and pass to your preferred JSON parser (e.g. cJSON):

```c
#include "ecewo.h"
#include "cJSON.h"

void create_user(Req *req, Res *res) {
  if (!req->body || req->body_len == 0) {
    send_text(res, 400, "Body required");
    return;
  }

  cJSON *json = cJSON_ParseWithLength((const char *)req->body, req->body_len);
  if (!json) {
    send_text(res, 400, "Invalid JSON");
    return;
  }

  const cJSON *name = cJSON_GetObjectItemCaseSensitive(json, "name");
  if (cJSON_IsString(name)) {
    printf("Name: %s\n", name->valuestring);
  }

  cJSON_Delete(json);
  send_text(res, 200, "Success!");
}
```

### Binary body

For binary data (e.g. `application/octet-stream`), use `req->body` and `req->body_len` directly.

```c
#include "ecewo.h"

void upload(Req *req, Res *res) {
  if (!req->body || req->body_len == 0) {
    send_text(res, 400, "Body required");
    return;
  }

  process_binary(req->body, req->body_len);
  send_text(res, 200, "Uploaded");
}
```

## Request Params

Let’s take a specific user by params. We can access the params using the `get_param(req, "slug");` function. Let’s write a handler that gives us the `slug`:

```c
#include "ecewo.h"
#include <stdio.h>

void send_params(Req *req, Res *res) {
  const char *user_id = get_param(req, "id");
  const char *action = get_param(req, "action");

  if (user_id == NULL || action == NULL) {
    send_text(res, 400, "Missing parameter");
    return;
  }

  send_text(res, 200, "Success!");
}

int main(void) {
  if (server_init() != 0) {
    fprintf(stderr, "Failed to initialize server\n");
    return 1;
  }

  // Route with parameters
  get("/users/:id/:action", user_handler);
  // Matches: /users/123/edit, /users/456/delete, etc.

  if (server_listen(3000) != 0) {
    fprintf(stderr, "Failed to start server\n");
    return 1;
  }

  server_run();
  return 0;
}
```

Compile the program and send a request to:

```
http://localhost:3000/send-params/testslug
```

Server will send us `testslug` response.

## Request Query

Just like the params, we can use `get_query(req, "query");` to get the query params. Let’s rewrite a handler using query:

```c
#include "ecewo.h"
#include <stdio.h>

void print_query(Req *req, Res *res) {
  const char *name = get_query(req, "name");
  const char *surname = get_query(req, "surname");

  if (name == NULL || surname == NULL) {
    send_text(res, 400, "Missing required parameter.");
    return;
  }

  printf("Name: %s Surname: %s\n", name, surname);

  send_text(res, 200, "Success!");
}

int main(void) {
  if (server_init() != 0) {
    fprintf(stderr, "Failed to initialize server\n");
    return 1;
  }

  get("/print-query", print_query);

  if (server_listen(3000) != 0) {
    fprintf(stderr, "Failed to start server\n");
    return 1;
  }

  server_run();
  return 0;
}
```

Let’s compile the program and send a request to:

```
http://localhost:3000/print-query?name=john&surname=doe
```

We’ll receive a `Success!` response and the query strings will be printed in the console:

```
Name: john Surname: doe
```

## Request Headers

Just like params and query, we can also access request headers using the `get_header(req, "header");` function. ecewo also provides different functions for authorization and session-based authentication. However, if you simply want to access a specific item in `req->headers`, you can do so directly.

Typically, a standard `GET` request with POSTMAN have some headers like:

```json
{
  "User-Agent": "PostmanRuntime/7.43.3",
  "Accept": "*/*",
  "Postman-Token": "9b1c7dda-27f9-471a-9cdd-bfaf0d5b56a1",
  "Host": "localhost:3000",
  "Accept-Encoding": "gzip, deflate, br",
  "Connection": "keep-alive"
}
```

Let’s say, we need the `User-Agent` header:

```c
#include "ecewo.h"
#include <stdio.h>

void get_user_agent(Req *req, Res *res) {
  const char *user_agent = get_header(req, "User-Agent");

  if (user_agent == NULL) {
    send_text(res, 400, "Missing required parameter.");
    return;
  }

  send_text(res, 200, user_agent);
}

int main(void) {
  if (server_init() != 0) {
    fprintf(stderr, "Failed to initialize server\n");
    return 1;
  }

  // Register the route
  get("/header", get_user_agent);

  if (server_listen(3000) != 0) {
    fprintf(stderr, "Failed to start server\n");
    return 1;
  }

  server_run();
  return 0;
}
```

Let’s send a request via POSTMAN:

```
http://localhost:3000/header
```

The result will be something like this:

```
User Agent: PostmanRuntime/7.43.3
```

## Request Timeout

You can set a timeout for a specific request using the `request_timeout()` function. This function applies only to the route in which it is called.

To configure a default global request timeout, see the [REQUEST_TIMEOUT_MS](./09.configurations.md#request_timeout_ms) in configurations. `request_timeout()` function overrides the default global timeout.

```c
int request_timeout(Res *res, uint64_t timeout_ms);
```

**Parameters:**
- `res`: Response object
- `timeout_ms`: Timeout in milliseconds

**Returns:**
- `0` on success
- `-1` on error

**Behavior:**
- Can be called multiple times to restart/update timeout
- Automatically cancelled when response is sent
- Closes connection if timeout expires

### Usage

#### No Timeout (Default)

```c
void fast_handler(Req *req, Res *res) {
  // No timeout - runs until complete
  send_text(res, OK, "Fast response");
}
```

#### Explicit Timeout

```c
void slow_handler(Req *req, Res *res) {
  // Set 30-second timeout
  request_timeout(res, 30000);
  
  do_slow_database_query();
  send_json(res, OK, results);
}
```

#### Reset/Renew Timeout

Call it multiple times to reset or renew the request timeout.

```c
void slow_handler(Req *req, Res *res) {
  // Set 30 second timeout
  request_timeout(res, 30000);
  
  run_db_query();

  // Renew the timeout to 60 second
  request_timeout(res, 60000);

  run_another_db_query();

  send_json(res, OK, results);
}
```
